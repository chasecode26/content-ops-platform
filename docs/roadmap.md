# 开发顺序

## 阶段 1：母稿工作流（公众号闭环）

目标：打通「母稿 -> 公众号派生 -> 草稿推送」

### 任务清单

1. 初始化 monorepo 与基础目录（pnpm workspace + Prisma 数据模型）
2. 母稿数据模型（master_draft 表：标题、正文、状态机、版本快照）
3. 建立母稿、模板、平台账号、发布任务 API
4. 母稿创建 API（导入 Markdown 入口）
5. 实现微信 HTML 渲染器 v1
6. 微信公众号适配器 v1
7. 创作台 Web 页面 v1（母稿编辑 + 预览 + 确认）
8. 打通草稿创建与失败日志记录

### 交付标准

- 能通过 Web 创建一篇母稿
- 母稿确认后能自动生成公众号派生版本
- 能选一个渲染主题并生成稳定 HTML 预览
- 能推送到公众号草稿箱
- 能查看母稿 -> 派生版本 -> 发布状态的完整链路

## 阶段 2：模板系统

目标：建立「生成模板」+「渲染主题」两层模板体系

### 任务清单

1. 生成模板数据模型（Prompt 模板 + 风格参数矩阵）
2. 渲染主题系统完善（CSS Token 系统、多主题切换）
3. ai-orchestrator 接入（AI Provider 路由、风格变体生成）
4. template-engine 模块整合
5. 模板中心 Web 页面（模板列表、在线编辑、实时预览）
6. 模板-主题组合推荐

## 阶段 3：平台派生版本

目标：从母稿自动派生多平台内容变体

### 任务清单

1. platform-variant-service 规则引擎
2. 小红书图文卡片格式派生
3. CSDN Markdown 博文格式派生
4. 平台派生版本管理 UI
5. 一稿多发任务编排

## 阶段 4：聊天入口

目标：Skill / Agent 对话式创作链路完整打通

### 任务清单

1. skill-gateway 项目初始化
2. 自然语言 -> 母稿生成链路
3. 预览返回 + 快速确认
4. OpenClaw Agent 对接
5. Web 创作台增加聊天入口引导

## 阶段 5：小红书桥接

目标：小红书图文/笔记自动发布

### 任务清单

1. Playwright 浏览器自动化桥接
2. 小红书图文卡片格式适配
3. 登录态持久化与安全
4. 发布状态回写与失败重试

## 阶段 6：CSDN 发布

目标：CSDN 博文自动发布

### 任务清单

1. CSDN API 对接（Markdown 直出格式）
2. CSDN 适配器实现
3. 标签、分类、封面图映射
4. 发布状态回写

## 推荐迭代节奏

### 第 1 周

- 仓库初始化
- 数据模型（含母稿）
- 基础 API

### 第 2 周

- 母稿创建（Markdown 导入）
- 微信 HTML 渲染器
- 创作台页面 v1

### 第 3 周

- 微信公众号适配器
- 草稿推送
- 预览和日志

### 第 4 周

- 生成模板系统
- 模板中心
- ai-orchestrator 接入

## 优先级原则

- **先打通母稿闭环**：母稿创建 -> 确认 -> 派生 -> 推送，再扩交互方式
- **母稿是一等公民**：所有平台内容从母稿派生，不再独立创建
- **模板分层设计**：生成模板管风格，渲染主题管排版，二者解耦
- **聊天与 Web 并重**：快速出稿走聊天，精修走 Web
- 先做草稿，再做自动发布
- 先保留人工确认，再逐步增加自动化比例
- 单平台先行：公众号闭环打通后，再扩展小红书、CSDN
