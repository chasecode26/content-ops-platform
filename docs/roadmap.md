# 开发顺序

## 阶段 1：最小可用版本

目标：打通 `Markdown -> 统一主题 -> 微信公众号草稿箱`

### 任务清单

1. 初始化 monorepo 与基础目录
2. 建立 PostgreSQL + Prisma 数据模型
3. 建立内容、主题、平台账号、发布任务 API
4. 实现 Markdown 导入与版本保存
5. 实现微信 HTML 渲染器 v1
6. 实现微信公众号适配器 v1
7. 增加预览页面与人工确认关口
8. 打通草稿创建与失败日志记录

### 交付标准

- 能导入一篇 Markdown
- 能选一个主题并生成稳定 HTML
- 能推送到公众号草稿箱
- 能看到草稿记录和失败原因

## 阶段 2：内容生产能力增强

目标：让内容不止能发，还能自动准备

### 任务清单

1. 增加 URL/RSS/关键词抓取入口
2. 增加正文提取与结构化清洗
3. 增加 AI 生成、改写、摘要、标题优化
4. 增加多版本内容管理
5. 增加封面文案和摘要生成

## 阶段 3：工作流自动化

目标：把单次操作变成可复用流程

### 任务清单

1. 建立 workflow 模板
2. 支持定时抓取、定时生成、定时推草稿
3. 增加人工审核节点
4. 增加失败重试和超时告警

## 阶段 4：多平台分发

目标：同一份内容适配多个平台

### 任务清单

1. 接入小红书桥接适配器
2. 接入抖音适配器
3. 建立平台专属内容变体生成
4. 增加一稿多发任务中心

## 阶段 5：运营数据回收

目标：让系统能根据结果反哺生产

### 任务清单

1. 拉取平台发布状态
2. 回收阅读、点赞、收藏、评论等指标
3. 建立内容效果看板
4. 为后续选题和改写提供依据

## 推荐迭代节奏

### 第 1 周

- 仓库初始化
- 数据模型
- 基础 API

### 第 2 周

- Markdown 导入
- 微信 HTML 渲染器
- 主题系统 v1

### 第 3 周

- 微信公众号适配器
- 草稿推送
- 预览和日志

### 第 4 周

- URL/RSS 抓取
- AI 改写
- 多版本管理

## 优先级原则

- 先打通单平台闭环，再扩平台
- 先做草稿，再做自动发布
- 先做统一内容模型，再做平台特化
- 先保留人工确认，再逐步增加自动化比例
