# Content Ops Platform Design

## 1. 设计概述

### 1.1 目标

建设一套本地优先、可持续扩展的内容生产与分发平台，先打通微信公众号长文排版与草稿推送，再逐步接入自动抓取、AI 写作、多平台发布与效果追踪。

### 1.2 非目标

- 第一阶段不做多人协作审批流
- 第一阶段不做全自动直接发布
- 第一阶段不做复杂推荐和热点分析
- 第一阶段不做全量视频生产工作流

## 2. 总体架构

```text
Content Input
  -> Source Connector
  -> Content Normalizer
  -> AI Writing Pipeline
  -> Theme Renderer
  -> Platform Adapter
  -> Draft / Publish Orchestrator
  -> Status Tracking & Audit
```

### 2.1 系统组件

- `web-admin`：管理后台，负责导入内容、主题选择、预览、人工确认、任务追踪
- `api-server`：统一对外 API，管理内容、主题、平台账号、发布任务
- `worker`：处理异步任务，包括渲染、抓取、AI 生成、平台推送
- `render-engine`：负责 Markdown AST 解析、主题化渲染和平台格式转换
- `adapter-sdk`：定义平台适配器标准接口
- `workflow-kernel`：编排抓取、生成、审核、渲染、推送等任务链

## 3. 关键业务流

### 3.1 MVP：Markdown 到微信公众号草稿箱

1. 用户上传 Markdown 或选择已有稿件
2. 系统解析 frontmatter 与正文结构
3. 用户选择主题风格
4. 系统渲染为公众号 HTML
5. 用户预览并确认
6. 系统上传正文图片与封面资源
7. 系统调用公众号草稿接口创建草稿
8. 系统保存草稿 ID、HTML 快照、上传日志

### 3.2 Phase 2：抓取到推草稿

1. 用户输入 URL、RSS 或关键词
2. Source Connector 抓取原始内容
3. Content Normalizer 提炼标题、摘要、正文、图片
4. AI Writing Pipeline 执行重写、扩写、摘要、标题生成
5. 用户确认内容版本
6. 渲染并推送到目标平台草稿

### 3.3 Phase 3：多平台分发

1. 一篇标准化内容进入分发中心
2. 按平台生成专属内容形态
3. 调用各平台适配器创建草稿或直接发布
4. 回写发布状态、平台 postId 与失败原因

## 4. 关键设计决策

| 决策 | 方案 | 理由 |
|------|------|------|
| 后端语言 | Node.js + TypeScript | 统一前后端语言，便于内容处理与自动化 |
| 服务结构 | 先模块化单体，后续再拆服务 | 当前优先效率，避免过早微服务 |
| 渲染方式 | Markdown AST -> Theme Renderer | 比字符串模板更稳定、可维护 |
| 发布方式 | 平台 Adapter 抽象 | 平台差异大，必须隔离 |
| 小红书接入 | 初期桥接独立自动化 Worker | 降低主系统复杂度，便于替换 |
| 任务执行 | BullMQ 异步任务 | 支持重试、延迟、并发控制 |
| 人工确认 | 推送前保留 review gate | 降低误发布风险 |

## 5. 部署模型

### 5.1 本地开发

- `web-admin` 本地启动
- `api-server` 本地启动
- PostgreSQL 与 Redis 本地容器运行
- `worker` 独立进程运行
- Playwright 浏览器持久化目录本地保存

### 5.2 轻量生产

- 单机 Docker Compose
- PostgreSQL
- Redis
- API + Worker
- 定期备份数据库与内容资源

## 6. 安全考量

### 6.1 敏感信息

- 平台账号密钥统一存储在数据库加密字段中
- 浏览器自动化登录态保存在独立安全目录
- 不在 Markdown、主题或任务参数中记录 token

### 6.2 发布安全

- 默认先创建草稿，不自动发布
- 平台推送保留人工确认开关
- 每次发布保留完整审计日志与 payload 摘要

### 6.3 内容安全

- 抓取内容进入 AI 管线前先做清洗
- 渲染阶段统一过滤危险 HTML
- 图片下载与外链引入要限制来源与大小

## 7. 已知限制

- 微信公众号的素材上传、封面图与正文图片链路较繁琐
- 小红书接入初期依赖浏览器自动化，稳定性受页面结构影响
- 多平台内容模型差异大，不能指望一份 HTML 直接复用
